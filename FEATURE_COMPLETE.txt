When you create a new version branch, the script now:

1. ✅ Asks you to classify the release as:
   • (stable)       - Production-ready
   • (experimental) - Beta/testing
   • (unstable)     - Development/WIP

2. ✅ Includes the stability in the branch name:
   • Before: 1.1.2
   • After:  1.1.2(stable), 1.1.2(experimental), or 1.1.2(unstable)

3. ✅ Updates version files with stability info:
   • VERSION file: 1.1.2(experimental)
   • __version__: "1.1.2(experimental)"
   • __stability__: "experimental"
   • __version_number__: "1.1.2"

4. ✅ Creates git branches with full names:
   git checkout -b 1.1.2(experimental)

# Check if production-ready
if __stability__ == "stable":
    print("This is a stable release")
else:
    print(f"⚠️  This is a {__stability__} release")

# Version gating
if __stability__ in ["experimental", "unstable"]:
    enable_beta_features()
    enable_debug_logging()

📋 STABILITY LEVEL GUIDE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🟢 (stable)
  ✓ Production-ready
  ✓ Feature complete
  ✓ Thoroughly tested
  ✓ No known critical bugs
  ✓ Safe for general users
  Example: 1.1(stable), 1.2(stable)

🟡 (experimental)
  • Beta/testing phase
  • New features under test
  • Feedback welcome
  • May have minor issues
  • Not for production use
  Example: 1.1.3(experimental), 1.2(experimental)

🔴 (unstable)
  ⚠ Work-in-progress
  ⚠ Alpha/preview
  ⚠ Frequent changes
  ⚠ Not recommended for normal use
  ⚠ Development/testing only
  Example: 1.1.5(unstable), 2.0(unstable)
