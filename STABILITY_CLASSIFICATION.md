# ✨ NEW FEATURE: Version Stability Classification

**Added**: April 27, 2026

## What's New?

The version management system now supports **release stability levels** for each version branch!

When you create a new version, you'll be asked to classify it as:
- **(stable)** - Production-ready
- **(experimental)** - Beta/testing
- **(unstable)** - Development/work-in-progress

## Examples

### Before (v0.1)
```
Branches: 1.1, 1.1.2, 1.1.3, 1.2
```

### Now (v0.2)
```
Branches: 1.1(stable), 1.1.2(stable), 1.1.3(experimental), 1.1.4(unstable), 1.2(experimental)
```


## Version File Updates

### VERSION file
```
1.1.2(experimental)
```

### core/__version__.py
```python
"""
Akira Forge Version Information
Auto-generated by create_version_branch.py
"""

__version__ = "1.1.2(experimental)"
__version_number__ = "1.1.2"
__stability__ = "experimental"
__version_info__ = (1, 1, 2)
```

## Using Version Info in Code

```python
from core.__version__ import __version__, __version_number__, __stability__

# Full version with stability
print(__version__)           # "1.1.2(experimental)"

# Version number only
print(__version_number__)    # "1.1.2"

# Stability level
print(__stability__)         # "experimental"

# Version tuple
print(__version_info__)      # (1, 1, 2)

# Example: Show warning for non-stable versions
if __stability__ != "stable":
    print(f"⚠️  Running {__stability__} build!")
```

## Stability Meanings

### (stable)
- **Status**: Production-ready
- **When to use**: 
  - Feature complete
  - Thoroughly tested
  - Ready for end users
  - No known critical bugs
- **Examples**: 1.1(stable), 1.2(stable), 2.0(stable)

### (experimental)
- **Status**: Beta/testing
- **When to use**:
  - New features being tested
  - Seeking user feedback
  - Not for production use
  - May have breaking changes
- **Examples**: 1.1.3(experimental), 1.2(experimental)

### (unstable)
- **Status**: Work-in-progress
- **When to use**:
  - Active development
  - Alpha/preview releases
  - Not recommended for general use
  - Frequent changes expected
- **Examples**: 1.1.5(unstable), 2.0(unstable)

## Git Branch Naming

All branch names now include the stability level:

```
git branch
* 1.1.2(experimental)
  1.1(stable)
  1.1.3(stable)
```

## Push to GitHub

Push with the full version name:

```bash
git push -u origin 1.1.2(experimental)
git push -u origin 1.1.2(stable)
```

## Filter by Stability

Find branches by stability:

```bash
# Find all stable branches
git branch -a | grep "(stable)"

# Find all experimental
git branch -a | grep "(experimental)"

# Find all unstable
git branch -a | grep "(unstable)"
```
